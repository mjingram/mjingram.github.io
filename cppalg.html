Access Level Problem

In many computer systems and networks, different users are granted different levels of access to different resources. In this case, there is a integer vector of "user_rights", indicating the privilege level of each user to use some system resource. There is also "int min_permission", which is the minimum permission a user must have to use this resource. This algorithm returns a string indicating which users can and cannot access this resource. Each character in the return value corresponds to the element of users with the same index. 'A' indicates the user is allowed access, while 'D' indicates the user is denied access.

    Constraints
  • user_rights will contain between 0 and 50 elements (0 ≤ user_rights.size() ≤ 50), inclusive.
  • Each element of user_rights will be between 0 and 100, inclusive.
  • min_permission will be between 0 and 100, inclusive.

#include <string>
#include <vector>
#include <fstream>
#include <iostream>
using namespacestd;

string can_access(vector<int> &user_rights, int min_permission) {
  string access="";
    for (int x: user_rights){
      if(x>=min_permission){
        access.push_back('A');
      }
      else{
        access.push_back('D');
      }
      }
      return access;
}

CG Ratio Problem

Regions of a DNA genome that code for a protein have a relatively high ratio of CG content. Given a strand of DNA, this algorithm determines the percentage of cytosine and guanine nucleotides present in the genome. The method returns the percentage (between 0.0 and 1.0) of 'c' (cytosine) and 'g' (guanine) in the genome. For example, if half of the nucleotides are either 'c' or 'g' the method would return 0.5. The DNA strand will contain only cytosine, guanine, thymine, and adenine, represented by the characters 'c', 'g', 't', and 'a', respectively.

    Constraints
  • The string dna contains only the characters 'a', 'g', 't', 'c' (lowercase).
  • The string dna is between 0 and 1000 characters long (inclusive).
#include <string>
using namespacestd;

double ratio(string dna) {
    // fill in code here
    double counter;
    double percent;
    double length = dna.length();
    for (int i; i<=length; ++i){
        if(dna[i]=='g'){
            counter+=1.0;
        }
        else if (dna[i]=='c'){
            counter+=1.0;
        }
  }
  percent = counter/length;

  return percent;
  }

Common Count Problem

Two strings have a letter in common if it appears in both strings. In this function, the position of the letter is not important for it to be counted as common. Once a particular letter is counted as in common, it is not counted again. For example, an o appearing twice in one word but only once in the other counts only as one letter in common. However, an o appearing twice both words counts as two letters in common. This function takes two strings and returns the number of letters they have in common.

    Constraints
  • Strings a and b are the same length
  • Each string only contains characters in the range 'a'-'z'
#include <string>
using namespacestd;

int count (string a, string b) {

    int length = a.length();
    int counter = 0;
    for(int i = 0; i<=(length-1); ++i){
        for (int j =0; j<=length-1; ++j){
          if (a[i]==b[j]){
              b[j]='@';
              counter+=1;
              break;
          }
              }

            }
      return counter;

  }
